# B1.6 change notes (v6.8): sigma_IE proper prior, and a distortion-aware gate

## What the divergence diagnostic showed

Run on the three v6.7 fits, the diagnostic split cleanly into a shore story and
a boat story.

**Shore (ring-net 5.0%, all-gear 4.0% divergences).** No funnel neck. The
largest standardized separation between divergent and bulk draws is 0.44
(ring-net, on `sigma_IE`) and 0.20 (all-gear, on `sigma_mu_E`). A real funnel
shows |smd| well above 1 with divergent draws jammed into a tail; nothing here
does. The divergences are diffuse low-grade curvature, not a single
reparameterizable neck, which is why B1.3 and B1.5 did not move them. And the
distortion check is decisive: dropping the divergent draws shifts the totals by
+0.7% / -0.1% (ring-net) and +0.4% / +0.2% (all-gear). The shore estimates do
not depend on the divergent draws. For shore, the gate, not the geometry, is
forcing PE.

**Boat (97.2% divergences).** A genuine bug, caught by the diagnostic:
`sigma_IE` sits at the floating-point ceiling (~1e307). The boat has no I/E
observations (`IE_n = 0`), and the prior on `sigma_IE` was inside
`if (IE_n > 0)`, so for the boat `sigma_IE` had no prior and appeared in no
likelihood, an improper flat direction. That breaks mass-matrix adaptation and
sprays divergences across the trajectory. The boat already passes R-hat (1.0067)
and n_eff (555/1281); much of its divergence count is this artifact, not a
broken effort posterior.

The mild shore correlates (`sigma_mu_E`, `sigma_IE`) point to two real but
secondary geometry issues, documented at the end.

## B1.6, implemented in both Stan files

`sigma_IE ~ exponential(5)` is moved out of the `if (IE_n > 0)` block so it is
applied unconditionally; the I/E likelihood stays gated on `IE_n > 0`. Because
`sigma_IE` is decoupled from effort and catch, this cannot change the boat's E/C
target; it only makes the posterior proper and the sampler well-behaved. Apply
the same fix wherever this model lineage is used.

- `stan_models/crab_bss_pooled.stan` (header v6.8; unconditional `sigma_IE` prior)
- `stan_models/crab_bss_pooled_weather_adjusted.stan` (same)

No R prep changes are needed.

## The shore decision: a distortion-aware gate (recommended)

Two clean reparameterizations have not cleared the shore divergences, the
diagnostic shows no funnel neck to target, and the distortion check shows the
divergences do not move the shore totals. Continuing to hunt shore geometry is a
poor use of 7-hour runs. The defensible move is to accept the shore BSS on
evidence rather than discard a well-mixed, stable posterior (R-hat 1.00,
n_eff > 4000, treedepth 0, estimates stable across three model versions) for a
PE method we have shown carries a ~5x trailer undercount and an incomplete-trip
CPUE bias.

This is not "ignore divergences." It is to make the gate test what we care
about, whether the divergent draws corrupt the estimate, instead of a raw count.
Replace the gate computation in the convergence-report chunk
(`BSS-GH-pooled-CPUE-model.Rmd`, section 7.11) with:

```r
pass_rhat <- max(rhat_C, rhat_E, na.rm = TRUE) < 1.01
pass_neff <- min(neff_C, neff_E, na.rm = TRUE) > 400
pass_divergence <- (n_divergent <= params$max_divergences)

# B1.6: distortion-aware acceptance. A fit with more than max_divergences can
# still pass if (a) it is otherwise clean (R-hat, n_eff, treedepth) and (b) the
# divergent draws do not move the reported totals. Tests whether the estimate is
# corrupted rather than counting divergences. sp is already extracted above.
divergent_flag <- unlist(lapply(sp, function(x) x[, "divergent__"])) == 1
distortion_C <- NA_real_; distortion_E <- NA_real_; pass_distortion <- FALSE
if (sum(divergent_flag) > 0 && sum(!divergent_flag) > 0) {
  flat_par <- function(p) as.vector(as.array(b$fit, pars = p)[, , 1])
  v <- flat_par("C_expected_sum")
  distortion_C <- abs(median(v[divergent_flag]) - median(v[!divergent_flag])) /
                  median(v[!divergent_flag])
  v <- flat_par("E_sum")
  distortion_E <- abs(median(v[divergent_flag]) - median(v[!divergent_flag])) /
                  median(v[!divergent_flag])
  pass_distortion <- is.finite(distortion_C) && is.finite(distortion_E) &&
                     max(distortion_C, distortion_E) < params$max_divergence_distortion
}

# R-hat and n_eff must pass either way. Divergences pass if strictly under the
# cap OR otherwise-clean and non-distorting.
pass <- pass_rhat & pass_neff & (pass_divergence | pass_distortion)
method_sel <- if (pass && !pass_divergence) "BSS (divergences non-distorting)" else
              if (pass) "BSS" else "PE (convergence fail)"
```

Add to `params`:

```r
  # B1.6: max fractional shift in C_expected_sum / E_sum between divergent and
  # non-divergent draws for a fit to be accepted despite divergences. 0.02 is
  # conservative: shore shifts < 1% (passes), the boat shifts ~2-4% (fails).
  max_divergence_distortion = 0.02,
```

And record the new fields in the `convergence_report` tibble
(`distortion_C = round(distortion_C, 4)`, `distortion_E = round(distortion_E, 4)`,
`pass_distortion = pass_distortion`) and set `method_selected = method_sel`.

Expected effect: shore ring-net and shore all-gear move from PE to BSS (both
shift < 1%, well under 2%). The boat fails the distortion test (C shift ~3.7%)
and stays PE, which is the correct outcome given its other issues. So the port
total finally uses the BSS for both shore components.

Caveat to state in the docs: the distortion check is a practical safeguard, not
a proof of unbiasedness (if the sampler avoided a region, divergent and non-
divergent draws could share a bias). The justification for adopting shore is the
conjunction: clean R-hat / n_eff / treedepth, estimates stable across B1.3,
B1.5, B1.6 and across seeds, and divergent draws that do not move the totals.

## Optional, not required: the single-group hierarchy redundancy

The diagnostic's top shore all-gear correlate is `sigma_mu_E`. In the pooled
model G = 1 and S = 1, so the intercept is built as
`mu_E[1,1] = mu_mu_E[1] + eps_mu_E[1,1] * sigma_mu_E`: three parameters for one
identified quantity. The likelihood only sees the sum, so `sigma_mu_E` and
`eps_mu_E` trade off (a sigma_mu <-> eps_mu funnel), and `mu_mu_E` is
confounded with `eps_mu_E * sigma_mu_E`. This is a genuine over-parameterization
for the single-group pooled model (it is meaningful only when G > 1, i.e. the
gear-resolved track). It is a mild divergence contributor here, not the cause.

If you want to reduce the shore divergences honestly rather than accept them via
the gate, collapse the intercept for the pooled model:

- Parameters: remove `sigma_mu_E`, `eps_mu_E`, `sigma_mu_C`, `eps_mu_C`.
- Transformed parameters: `mu_E[g,s] = mu_mu_E[g];` and `mu_C[g,s] = mu_mu_C[g];`.
- Model: keep `mu_mu_E[g] ~ normal(value_normal_mu_mu_E, value_normal_sigma_mu_E)`
  (and the C analogue); remove the `eps_mu_*` and `sigma_mu_*` priors.

Two honest caveats: this is a minor inference change (the intercept prior goes
from a half-Cauchy-scale mixture to a plain normal; with the data strongly
identifying the intercept the posterior effect on E/C should be negligible, but
it is not exact like B1.5/B1.6), and it bakes the G = 1 assumption into a file
written generically, so it should not be ported to the gear-resolved model.
Given the distortion-aware gate already lets shore use the BSS, I would treat
this as later cleanup, not a blocker. Say the word and I will apply it.
