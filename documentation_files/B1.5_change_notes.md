# B1.5 change notes (v6.7): marginalize the effort-count overdispersion

## What changed and why

The residual shore divergences that survived B1.3 (292 and 573 in the v6.6
pooled run, with otherwise clean R-hat and n_eff, and the 292-vs-1669 seed
instability) come from the per-observation effort overdispersion, not from the
AR scale priors I floated last turn.

In the model block, each effort count was

```
eps_E_H_obs[i] ~ gamma(r_E, r_E);                          // n_effort_obs of these
Gear_I[i] ~ poisson(lambda_E * eps_E_H_obs[i] * R_G);
T_I[i]    ~ poisson(lambda_E * eps_E_H_obs[Gear_n + i] * R_T);
```

`eps_E_H_obs` is a length-`n_effort_obs` vector of latent parameters whose
distribution depends on `r_E = 1 / sigma_r_E^2`. That is a centered,
high-dimensional funnel (311 parameters for shore all-gear, 52 for ring-net),
structurally the same problem B1.3 fixed for `omega_0`, with the neck at large
`r_E` / small `sigma_r`. B1.3 non-centered the AR path and `omega_0`; this
overdispersion block was the remaining centered piece, so it is where the
residual divergences live.

The fix is an exact analytic marginalization. A gamma-Poisson mixture with
`eps ~ Gamma(r, r)` integrates out to Negative Binomial 2:

```
Gear_I[i] ~ neg_binomial_2(lambda_E * R_G, r_E);
T_I[i]    ~ neg_binomial_2(lambda_E * R_T, r_E);
```

mean `lambda * R`, variance `lambda*R + (lambda*R)^2 / r_E`. `r_E` is unchanged,
so the overdispersion is identical. The marginalization is exact, so the
posterior over every reported quantity (E_sum, C_sum, C_expected_sum, B1, B2,
B1_C, R_G, R_T, L, ...) is the same up to Monte Carlo noise; the only thing
removed is the `n_effort_obs` nuisance parameters and their funnel.

Three reasons this is the right move rather than a prior swap:

1. It removes the actual funnel by construction, instead of reshaping a prior
   on `sigma_r` whose funnel neck is at small `sigma_r` (where a half-normal
   would not help and could pull toward the neck).
2. The interview catch likelihood already uses `neg_binomial_2(.., r_C)`, and
   the augmented file's `log_lik` already uses `neg_binomial_2(lambda*R, r_E)`
   for the effort counts (it documents the equivalence in its own header). The
   model block was the only place left on the conditional Poisson form. This
   change makes the three consistent.
3. It also resolves the conditional-vs-marginal LOO issue: with the per-
   observation random effects gone from the model, the model block and
   `log_lik` are now the same form, so the PSIS-LOO comparison is on solid
   ground (Vehtari, Gelman and Gabry 2017).

`n_effort_obs` is now an unused data field, retained only so the R prep does
not need to change. No R prep edits are required.

## Expected outcome and what to watch

- Shore divergences should drop sharply (this funnel held the bulk of them).
  If shore_ring_net and shore_all_gear fall to <= 5, both shore components pass
  the gate and use BSS instead of PE.
- The shore catch and effort medians should be statistically unchanged from
  v6.6, because the marginalization is exact. A large shift would indicate a
  transcription error, not a real effect; check that first.
- The boat is unchanged by this edit. Its non-convergence is weak
  identification from the trailer-count series, not this funnel, so expect it
  to stay on PE. Do not read a still-diverging boat as a failure of B1.5.
- If a few shore divergences remain after this, the next lever is the AR
  innovation scale (`sigma_eps`), but I expect the overdispersion marginalization
  to be sufficient on its own.
